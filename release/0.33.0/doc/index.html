YSH has 3 main sublanguages:

  • Command language, which now consistently uses { } for blocks.
    • Hay - Custom Languages for Unix Systems. Use Ruby-like blocks to declare data and interleaved code.
  • Word language.
    • Simple Word Evaluation. Written for shell experts.
  • Expression language on typed data.
    • Egg Expressions. A new regex syntax, abbreviated eggex.
    • YSH Regex API. Convenient and powerful.

Data Languages Avoid Ad-Hoc Parsing

YSH programs "talk about" these data languages, also called interchange formats or protocols. In-memory data structures are in service of data languages on the wire, not the other way around.

  • J8 Notation. An upgrade of JSON to bytes, strings, lines, and structured data.
